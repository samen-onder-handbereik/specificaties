# Glossary

## Uitkomst Overleg

**Status:** Werkbegrip -- kandidaat voor NL-SBB

De informatie die via de inzage-API beschikbaar wordt gesteld als resultaat van een casusoverleg.


## Betrokkene

**Status:** Werkbegrip -- kandidaat voor NL-SBB

De persoon waarop de Uitkomst Overleg betrekking heeft.


## Entity

**Status:** Technisch begrip

Een PROV-concept dat een informatieobject of ander identificeerbaar object representeert.


## Activity

**Status:** Technisch begrip

Een PROV-concept dat een proces of gebeurtenis beschrijft waardoor informatie ontstaat of wordt gebruikt.


## Agent

**Status:** Technisch begrip

Een PROV-concept dat de actor representeert die verantwoordelijk is voor een activiteit.


## Provenance

**Status:** Werkbegrip -- kandidaat voor NL-SBB

Informatie over de herkomst en het gebruik van gegevens.


## Identifier

**Status:** Technisch begrip

Een waarde waarmee een element binnen een model eenduidig kan worden onderscheiden.


## Businessidentifier

**Status:** Werkbegrip

Een identifier die betekenis heeft binnen het domeinmodel.


## Provenance identifier

**Status:** Werkbegrip

Een technische identifier waarmee een PROV-element uniek wordt aangeduid.


## Burgerservicenummer

**Status:** Werkbegrip

Een persoonsgebonden attribuut van de Entity Betrokkene dat gebruikt kan worden om de graph te bevragen.


## JSON-LD context

Een context waarin prefixen en termen worden gekoppeld aan volledige URI's.


## Namespace

Een URI-gebied waarin begrippen uniek worden geïdentificeerd.


## Prefix

Een verkorte aanduiding voor een namespace, bijvoorbeeld `prov` of `soh`.


## URI

Een unieke verwijzing naar een resource of begrip.

## CloudEvent

**Status:** Technisch begrip

Een gebeurtenisbericht volgens de CloudEvents-specificatie van de Cloud Native Computing Foundation (CNCF), bestaande uit context-attributen en een inhoudelijke payload.


## CloudEvent API

**Status:** Technisch begrip

Een API waarmee een aanbieder een CloudEvent aanbiedt voor verwerking.


## CloudEvent envelope

De verzameling standaard context-attributen van een CloudEvent die metadata over de gebeurtenis bevatten.


## JSON-LD payload

De gegevensinhoud van een CloudEvent die volgens JSON-LD is gestructureerd.


## Semantisch contract

De afspraken waarmee de betekenis van uitgewisselde gegevens eenduidig wordt vastgelegd.


## Respec

Een specificatiegenerator die onder andere automatische paragraafnummering verzorgt.


## JSON-LD type

Een classificatie van een resource binnen JSON-LD, vastgelegd via `@type`.


## Query-API

**Status:** Technisch begrip

Een API waarmee een deelnemer queries kan uitvoeren op gegevens die binnen een
samenwerkfunctie beschikbaar zijn gesteld.

Een Query-API is specifiek voor een samenwerkfunctie en wordt niet generiek
gedefinieerd. De beschikbare queries sluiten aan op de informatiebehoefte
binnen de betreffende samenwerkfunctie.




## Status-API

**Status:** Technisch begrip

Een API waarmee de status van een eerder aangeboden verwerking kan worden opgevraagd.



## Asynchrone interactie

**Status:** Technisch begrip

Een interactie waarbij het initiëren van een verzoek of gebeurtenis en de
verdere verwerking daarvan niet in dezelfde stap plaatsvinden.


## Generiek interactiepatroon

**Status:** Technisch begrip

Het patroon waarbij een interactie wordt geïnitieerd, geaccepteerd, vervolgens
asynchroon wordt verwerkt en waarbij de voortgang en/of het resultaat later
beschikbaar komt.



## Knowledge graph

**Status:** Technisch begrip

Een graaf waarin informatieobjecten, hun onderlinge relaties en
provenance-informatie worden vastgelegd zodat deze semantisch bevraagbaar zijn.
